# 📘 JD Edwards (JDE) — Complete Notes

---

## 1. What is JD Edwards (JDE)?

**JD Edwards (JDE)** is one of the world's leading **Enterprise Resource Planning (ERP)** software solutions, now owned and maintained by **Oracle Corporation**. It is a fully integrated suite of over **80+ application modules** that helps businesses manage and automate core functions like financials, supply chain, manufacturing, human resources, sales, and more — all from a single centralized platform.

> **In simple terms:** JDE is an enterprise-grade ERP system trusted by over **7,000 companies worldwide** to run their day-to-day and strategic business operations.

### Current Products (as of 2026)
| Product                           | Description                                        |
| --------------------------------- | -------------------------------------------------- |
| **JD Edwards EnterpriseOne (E1)** | Main web-based, modern ERP (most widely used)      |
| **JD Edwards World**              | Original AS/400-based legacy ERP (still supported) |

### 2.1 Founding
- Founded on **March 1, 1977** in **Denver, Colorado, USA**
- Started in a tiny office with a single copy machine they rented for **$105/month**
- Initially focused on developing **accounting software** on IBM green-screen terminals
- Early clients were small businesses needing mainframe accounting tools

## 3. Why Was JDE Created?

In the late 1970s, businesses using IBM minicomputers (System/34, System/36, System/38) had no integrated software to manage their supply chains, accounting, and operations together. JDE founders saw this gap and built accounting and ERP software specifically for **IBM minicomputers**, solving the critical problem of disconnected business systems.

**Key Problems JDE Solved:**
- No integrated business software for IBM platform users
- Manual, error-prone bookkeeping
- Disconnected inventory, finance, and production systems
- Lack of real-time visibility into operations

## 4. JDE Version History & Evolution

### 4.1 Phase 1 — JDE World (1980s–1990s)

- The **original JDE product** — built for **IBM AS/400 minicomputers**
- Used **green screen terminals** (character-based interface)
- Server-centric architecture — all processing happened on the mainframe
- Focused heavily on **accounting and supply chain**
- Covered all core ERP areas: Finance, Manufacturing, Distribution, HR
- Written in **RPG (Report Program Generator)** language

**JDE World Versions:**

| Version        | Notes                                                      |
| -------------- | ---------------------------------------------------------- |
| Early releases | Simply called "J.D. Edwards Software" with version numbers |
| A7.1           | First version to co-exist with OneWorld                    |
| A7.3           | Most widely deployed World version — very stable           |
| A8.1           | Not co-existent with OneWorld                              |
| **A9.4**       | Latest World version — Oracle recommends upgrading to this |

> 💡 Hundreds of global companies still run JDE World today.

---

### 4.2 Phase 2 — JDE OneWorld (1996–2003)

- In the mid-1990s, JDE recognized the industry shift from IBM mainframes to **client-server systems**
- The new product was internally codenamed **"Everest"** before it was released
- Launched in **late 1996** as **JD Edwards OneWorld**
- Revolutionary because it introduced:
  - **Graphical User Interface (GUI)** — replaced green screens
  - **CNC Architecture (Configurable Network Computing)** — ran on multiple OS and databases
  - Platform independence — could run on Windows, Unix, Linux, and more
  - A **fat client** model (software installed on user's workstation)
- For a transition period, both World and OneWorld could run in **"Co-Existence" mode** — sharing the same data schema

**OneWorld Versions:**

| Version               | Name                                      |
| --------------------- | ----------------------------------------- |
| B73.3.1               | Early OneWorld                            |
| Xe (pronounced "Ten") | Major release with expanded functionality |
| 8.0                   | Final OneWorld series before renaming     |

---

### 4.3 Phase 3 — JDE EnterpriseOne (2003–Present)

- When **PeopleSoft acquired JDE in 2003**, they blended the names "Enterprise" (from PeopleSoft Enterprise) and "OneWorld" → creating **"EnterpriseOne"**
- Introduced a **web-based thin client** — users no longer needed software installed locally; they used a browser
- When **Oracle acquired PeopleSoft in 2005**, Oracle continued EnterpriseOne's development and expanded it significantly
- Oracle committed to support E1 9.2 with **Premier Support through at least 2036**

**EnterpriseOne Versions:**

## 6. How Does JDE Work?

### 6.1 The Central Database Model

Like all ERP systems, JDE operates on a **single shared database** — all modules write and read from the same data source, ensuring no duplication and real-time visibility.

## 7. JDE Technical Architecture — CNC

### 7.1 What is CNC?

**CNC = Configurable Network Computing**

CNC is the proprietary technical architecture of JD Edwards. It is the "engine" that allows JDE to run on a vast range of hardware, operating systems, and databases without users needing to know anything about the underlying technology.

> **Analogy:** CNC is like the fuel system in a car — you don't need to understand how the injectors work to drive the car. JDE users just use the application; CNC handles everything behind the scenes.

### 7.2 What CNC Does
- Insulates business applications from the underlying hardware and OS
- Allows JDE to run on **Windows, Linux, IBM i, Solaris, AIX** simultaneously
- Supports **multiple databases** — Oracle DB, SQL Server, IBM DB2
- Manages where application logic runs (server vs. client)
- Enables distributed computing across a network of servers

### 7.3 Key CNC Concepts

| Concept                                | Description                                                                                                     |
| -------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| **Environment**                        | A collection of pointers telling JDE where data and objects are stored (e.g., Production, Development, Testing) |
| **Path Code**                          | Defines where application objects (code, reports, forms) are stored                                             |
| **OCM (Object Configuration Manager)** | Maps which database to use for each table and where logic should run                                            |
| **Package Build**                      | The process of compiling and deploying code changes to users                                                    |
| **ESU (Electronic Software Update)**   | A patch bundle from Oracle containing bug fixes (SARs)                                                          |
| **SAR (Software Action Request)**      | A tracked request for a bug fix or enhancement sent to Oracle                                                   |
| **ASU (Application Software Update)**  | Enhancement bundle for a specific functional area                                                               |
| **OMW (Object Management Workbench)**  | Change management tool for tracking and deploying object changes                                                |

### 7.4 JDE Server Types

| Server                  | Role                                                               |
| ----------------------- | ------------------------------------------------------------------ |
| **Enterprise Server**   | Main logic server where application processing runs                |
| **Deployment Server**   | Manages and deploys packages/updates to other servers              |
| **Web Server**          | Hosts the JDE web interface (HTML Server)                          |
| **Database Server**     | Stores all business data                                           |
| **AIS Server**          | Application Interface Services — enables orchestrations and mobile |
| **BI Publisher Server** | Generates formatted reports                                        |

### 7.5 JDE Development — Programming
JDE uses two types of programming:
1. **Event Rules (ER)** — A 4th generation wizard-based language developed by JDE for building forms and interactive applications
2. **Business Functions (BSFN)** — Written in **C/C++** language; these are compiled reusable code modules that execute business logic

## 9. JDE Unique Features & Tools

### 9.1 Orchestrator Framework
- A low-code/no-code automation tool built into JDE
- Allows users to create automated workflows (Orchestrations) without writing code
- Connects JDE to external systems via REST APIs
- Enables IoT integration — machines can send data directly into JDE
- Example: A sensor detects low oil level → automatically creates a maintenance work order in JDE

### 9.2 UX One
- A modern, role-based user interface within JDE
- Pre-built "Cafe One" pages personalized for specific job roles (e.g., CFO dashboard, warehouse manager view)
- Embedded analytics, watchlists, and KPIs

### 9.3 User Defined Objects (UDOs)
- Allows users to create and save personal/shared customizations without touching the source code
- Includes personal forms, saved queries, notifications, and dashboards

### 9.4 Mobile Applications
- JDE was the **first ERP to run all applications on Apple iPad**
- Mobile apps for field operations: work orders, time entry, inspections, approvals
- Offline capability — works without internet, syncs when connected

### 9.5 Business Services (BSSV)
- Web services exposed from JDE for integration with external systems
- Enables SOA (Service Oriented Architecture) integrations
- Used to connect JDE with portals, e-commerce platforms, and third-party tools

## 13. Advantages & Disadvantages of JDE

### 13.1 Advantages ✅
- Over 80 fully integrated modules in one suite
- Very strong for manufacturing and distribution industries
- Platform flexibility — runs on many OS, DB, and hardware combinations
- Mobile-ready and browser-based (no fat client needed in E1)
- Highly customizable without breaking upgrade paths (when done correctly)
- Strong Oracle support and continuous delivery model
- Long-term roadmap with Oracle backing
- One View Reporting gives real-time insights without a separate BI tool
- Orchestrator enables powerful automation and IoT integration
- Competitive total cost of ownership (TCO)

### 13.2 Disadvantages ❌
- CNC technical architecture is **complex** — requires specialized admins
- **Steeper learning curve** compared to some modern SaaS ERPs
- UI, while improved, is still considered less intuitive than SAP Fiori or Workday
- Limited native SaaS offering (not cloud-native like Oracle Fusion Cloud)
- Customizations can make upgrades painful if not managed carefully
- Limited presence in HR/Talent management compared to dedicated HR tools
- Smaller partner ecosystem compared to SAP
